# 架构决策记录（ADR）

轻量记录"容易日后被推翻重议"的决策：背景 → 约束 → 决策 → 由此产生的不变量。不全铺开，只记"为什么最终这么定"——避免下次有人（包括未来的自己）又把架构推倒重来。

## 何时新增 ADR

- 一项决策曾在版本间反复过（如先做 A 又改 B 又回到 A）。
- 决策的"为什么"无法从代码或 CHANGELOG 直接看出。
- 决策暗含一条**不变量**（改动它会破坏多处约定）。
