# 已有基础框架能力清单（模板 —— 请替换为你们平台的实际内容）

> 本文件是所有阶段 skill（尤其是 requirement-clarification 与 code）的**必读文件**，用于避免重复造轮子。请在正式接入本插件前，将下方内容替换为你们平台已有能力的真实清单，并在能力变化时同步维护。

## 已有能力清单

| 能力域 | 已提供的功能 | 复用方式 / 扩展点 | 禁止事项 |
|---|---|---|---|
| 登录鉴权 | 例：账号密码登录、Token鉴权、单点登录 | 例：新接口在Controller方法上加 `@RequiresAuth` 注解即可自动鉴权 | 禁止新建独立的登录/鉴权模块 |
| 用户/权限管理 | 例：用户增删改查、角色分配、菜单权限树 | 例：新增菜单项需按约定格式插入菜单表，并配置权限编码 | 禁止绕过权限树自行实现按钮级权限 |
| 系统管理 | 例：字典管理、日志审计、参数配置 | 例：新枚举值优先复用字典表，而非硬编码 | 禁止新建平行的配置管理机制 |
| （按需补充） | | | |

## 需求分析阶段的接入规则

- `requirement-clarification` skill 在产出需求规格前，必须对照本表，凡是已有能力覆盖的诉求，标注为"复用现有能力"，不生成新的 REQ 编号。

## 设计阶段的接入规则

- `design` skill 在设计涉及以上能力域的功能时，只描述"如何接入扩展点"，不重新设计整套机制。

## 编码阶段的接入规则

- `code` skill 直接复用已有的注解/组件/表结构，禁止重新实现。

---

**维护提示**：当基础框架新增能力（如新增SSO方式、新增审计维度）时，请同步更新本文件，否则后续需求分析可能重复生成已存在的功能。
